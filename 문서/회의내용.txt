1. 동작 프로세스에서 시간 단위 설정 
- 몇 초 이내에 프로세스를 실행할 지 시간을 설정하는 기능을 추가한다. 
- 총 횟수(유저수*반복횟수)를 설정한 시간으로 나누어 TPS를 측정한다.
- ex) 오전 1시 ~ 오전 6시까지 동작 실행

2. 시나리오 예약기능 - 특정 시간대에 트래픽이 몰려 성능저하가 일어나는지 판단 
- 단일 시나리오에 대한 예약 처리 기능을 구현한다.
- 원하는 시간을 지정하여 예약설정한 시나리오를 테스트할 수 있도록 구현한다.
- ex) 오전 1시에 '시나리오 1' 실행

3. 통계 그래프 x축 단위 조정
- 응답시간이 짧을 시, 초(sec) 사이에 ms 단위까지 추가하여 표기한다.
- 응답시간이 길어질 시, 1초 단위보다 범위를 넓게 표기(ex. 10초, 20초)한다.

4. 종합 집계 데이터 확인
- 통계조회페이지에서 차트 상단에 종합 집계 데이터를 표기한다.
- 종합 집계 데이터 : 시작시간, 종료시간, 전체수행횟수, 성공횟수, 실패횟수, 최대응답시간, 최소응답시간

5. 차트 페이지 구현
- 통계그래프 페이지를 추가하여 ITSK에서 표기하고자 하는 항목들을 그래프로 표현한다.
- 항목이 결정되지 않았기 때문에 ITSK에서 추후 원하는 자료를 보내주면 그 항목에 맞는 그래프를 추가하여 나타낸다.

1. 알 수 없는 언어로 올 때가 있다.  	인코딩 방식을 변경하여 확인하면 된다.
2. GET, POST 각각 언제 사용하는지? 	문서를 보고 판단 ex) 주소에 붙여 보내주세요 - GET, 오브젝트로 보내주세요 - POST
3. Timeout을 설정하는 이유?	요청을 보낸 후 설정한 Timeout 이내에 응답이 오지 못하면 return을 보내지 않는다.
4. Parameter 사용법	문서를 확인하여 문서에 있는 Param 값을 넣어준다. &로 값 구분 ex) &params[cttList][0][fee]=0
